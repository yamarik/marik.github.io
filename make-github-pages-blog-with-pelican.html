Make a Github Pages blog with Pelican

Posted on November 20, 2023 in Pelican

Install Pelican

In project’s folder:

pip install pelican

You can inslall Pelican with Sinaptic package manager (recommended).

Create a new folder for your project and open a console in it. Let's create default Simple theme:

pelican-quickstart

During the pelican-quickstart process you need to say "yes":

Do you want to upload your website using Github Pages? Say “y”

Do you want to automate site generation and publication? Say "y"

If you answered “yes” to second question, a tasks.py and Makefile will be generated in the root of your project. These files should be customized to fit your particular needs and usage patterns. Using them you can easily view and publish your site.

Creating blog content in Markdown

Open your favorite text editor (in my case vi) and navigate to project's root folder. Let's create the folder for our content: static pages and blog posts.

mkdir content
cd content

Create first post: 

vi your-first-post.md

You can add static pages:

mkdir pages
cd pages
vi about.md
vi contacts.md

All files have metadata:

  1. Title: title of your page/post.
  2. Date: 2023-02-26
  3. Category: Linux
  4. Slug: your page/post URL.Ex: https://yamarik.github.io/linux-disk-backup-using-clonezilla.html. Here slug is: linux-disk-backup-using-clonezilla

It's time to view our Simple Theme. Navigate to project's root folder and run:

make devserver

This step create OUTPUT folder with site files in root of your project. This command also tell Pelican to watch for your modifications, instead of manually re-running it every time you want to see your changes. In Linux it simultaneously both auto-regenerate and serve the output at http://127.0.0.1:8000/ or http://localhost:8000 (depends on your theme settings).

Change default Simple theme

Choose your theme

You can find pelican themes here or here

Clone your theme.

git clone https://github.com/alexandrevicenzi/Flex.git

Downolad theme.zip, unpack it and copy in your project's root folder. Here run:

pelican-themes -i Flex-master

Verify the installation

pelican-themes --list –verbose

It’s the path to your new theme

Configure blog to use new theme

For this, just open your pelicanconf.py available in your root folder(not in theFlex-master folder!) and define the path to your new theme by adding the following statement at the end of pelicanconf.py file:

THEME = '/usr/lib/python3/dist-packages/pelican/themes/Flex-master'

Rebuild and run the site locally

make devserver

It wil generate OUTPUT folder in your project with static site pages. It also tell Pelican to watch for your modifications, instead of manually re-running it every time you want to see your changes.

Go to http://127.0.0.1:8000 or http://localhost:8000/ in your browser to view your blog. If you want to stop development server run:

Ctrl+C

But now you still can't see all theme features. In project's root folder find Makefile, open it in your favorit editor and put correct paths to theme's content, output folder and settings. 

BASEDIR=$(CURDIR)
INPUTDIR=$(BASEDIR)/Flex-master/docs/content
OUTPUTDIR=$(BASEDIR)/Flex-master/output
CONFFILE=$(BASEDIR)/Flex-master/docs/pelicanconf.py
PUBLISHCONF=$(BASEDIR)/Flex-master/docs/publishconf.py

Then in project's root folder open tasks.py and change values in variables:

SETTINGS_FILE_BASE = 'Flex-master/docs/pelicanconf.py'
...
'settings_publish': 'Flex-master/docs/publishconf.py'

If you will have problems with rendering pages because of CACHE try open another page of site and then return to previous page again. IF you still have problems with CACHE edit pelicanconf.py in theme's folder: 

LOAD_CONTENT_CACHE = False
AUTORELOAD_IGNORE_CACHE = True

You can add default pagination count. You need put "0" here to show tags button on the main page.

DEFAULT_PAGINATION = 0

Let's create static pages in Flex-master/docs/content/pages/ and blog posts in Flex-master/docs/content/articles/ folder. For this task you need install markdown via pip or Sinaptic.

You can make changes in Flex-master/templates/partial/ folder if you wish or edit base.html file in template folder.

When you are ready to deploy your site for production change SITEURL in theme's pelicanconf.py from: 

SITEURL = "http://localhost:8000"

to

SITEURL = "https://your_site.github.io"

and run again:

make devserver

Push site on github

Create repository for your Github Pages on Github. Clone your site folder:

git clone https://github.com/yamarik/yamarik.github.io.git

In cloned folder:

git init .

You should see .git folder. 

git checkout -b main

Copy files in OUTPUT folder of your theme (in Flex-master folder) and paste it into cloned site folder from github. 

git add .

git commit -m "initial commit"

git push https://your_token@github.com/yamarik/yamarik.github.io.git

You are all done!

Note! Since 2022 you need to receive token instead of login and password to push site on github.
